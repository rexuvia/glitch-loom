## Multi-Agent Cron Pipeline Discussion

Sky asked about spawning agents that can set multiple cron jobs which will each spawn new agents with distinct goals later. Confirmed this is fully supported in OpenClaw architecture:

1. Orchestrator agent can use `cron` tool to create scheduled `agentTurn` jobs
2. Each cron job can have `sessionTarget: "isolated"` and distinct `message` payloads
3. When cron triggers, Gateway automatically spawns isolated sub-agents with those goals
4. Delivery can be targeted to specific channels (e.g., `telegram:6839797771`)

**Example payload schema:**
```json
{
  "schedule": { "kind": "at", "at": "2026-03-05T10:00:00Z" },
  "payload": {
    "kind": "agentTurn",
    "message": "Specific goal for this agent..."
  },
  "sessionTarget": "isolated",
  "delivery": {
    "mode": "announce",
    "channel": "telegram",
    "to": "6839797771"
  }
}
```
