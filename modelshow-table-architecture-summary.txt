## Architecture Highlights:

### Data Flow
1. ModelShow skill writes JSON results to workspace
2. Post-save hook runs `update_modelshow_index.py` to copy to web directory
3. Script updates `index.json` with metadata for frontend
4. Frontend polls `index.json` every 30s, renders table with sorting/filtering
5. Users can expand rows to see full comparison details

### Frontend Design
- Single Vue component with reactive state
- Polling for real-time updates
- Sortable columns (timestamp, models, winner, score, duration)
- Filter by model and date range
- Expandable detail view with judge analysis and responses
- Matches existing rexuvia.com design (dark theme, starry background)

### Backend Integration
- No server-side API needed; static JSON files served by nginx
- Index generation script handles retention and pruning
- Atomic writes prevent corruption
- Optional cron job for periodic full sync

### Deployment
- Integration with existing `rebuild.sh` via `prebuild` npm script
- Files placed in `public/modelshow-results/` for static serving
- No changes to nginx configuration needed
