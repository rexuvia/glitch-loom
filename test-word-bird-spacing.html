Word Bird Obstacle Spacing Analysis

Current Constants:

  • SPACING_BASE = 460
  • SPACING_MIN = 230
  • SPACING_DEC = 3
  • BOX_WIDTH = 155
  • Bird starting x = 110

Problem Analysis:

Issue #1: On wider screens, first wall starts at canvas.width + 100

If canvas.width = 1920 (typical desktop), first wall at x = 2020

Bird at x = 110, needs to travel 1910 pixels to reach first wall

At SPEED_BASE = 1.0, that's 1910 frames ≈ 31.8 seconds at 60fps!

Issue #2: On taller screens, vertical spacing might be too large

Box height = canvas.height / NUM_BOXES (where NUM_BOXES = 5)

If canvas.height = 1080, box height = 216 pixels

Bird radius = 18, so bird needs to move 180 pixels vertically between boxes

Proposed Solution:

1. Dynamic initial wall placement:

  • First wall should be placed based on screen width, not at canvas.width + 100
  • Maybe place first wall at min(canvas.width * 0.7, 800) pixels from left

2. Responsive spacing:

  • Base spacing should scale with screen dimensions
  • SPACING_BASE = Math.min(460, canvas.width * 0.3)
  • SPACING_MIN = Math.min(230, canvas.width * 0.15)

3. Screen-adaptive box sizing:

  • Box height should have minimum/maximum limits
  • Maybe: boxHeight = Math.min(Math.max(canvas.height / 6, 120), 200)
